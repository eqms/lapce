<objective>
Eliminate the remaining two crash sites in the plugin subsystem: CRASH-03 (DAP stdio-capture
`.unwrap()` in `dap.rs:104,105`) and CRASH-04 (zstd decoder `.unwrap()` in `plugin/mod.rs:1590`).
Both fixes use `?` propagation; both require their errors to surface as user-visible notifications
via `CoreNotification::ShowMessage` (CRASH-03 via catalog.rs, CRASH-04 via existing VoltInstalling
error path). Ships regression tests for both per TEST-01.

Depends on Plan 01 (03-01): the CRASH-04 fix requires the Bytes-then-Cursor pattern to be in place
(async `reqwest::Response` does not implement `Read`); Plan 01 migrates the response handling in
`download_volt` to use `Cursor::new(body)` with the `.unwrap()` temporarily preserved. This plan
replaces that `.unwrap()` with `?` + error message.

Output: Updated `dap.rs`, `catalog.rs`, `plugin/mod.rs` with crash fixes and regression tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Plugin archive bytes (from S3/CDN) → zstd::Decoder | Archive body is untrusted; may be malformed, truncated, or adversarially crafted |
| DAP plugin process spawn → stdio handles | Process may not have piped stdio depending on Command configuration; None handle is a trust boundary |
| Plugin archive → archive.unpack() | Even after fixing CRASH-04, the unpack() call may contain path-traversal entries — SEC-04 (Phase 4) scope |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Denial of Service | zstd::Decoder::new on malformed archive | mitigate | .map_err(|e| anyhow!(...))? stops the panic and returns Err; decompression bomb risk remains (archive.unpack reads all content) — Phase 4 adds size limits |
| T-03-11 | Elevation of Privilege | archive.unpack(&plugin_dir) after zstd decode | accept | Path-traversal guard is explicitly SEC-04 / Phase 4; CRASH-04 only stops the panic, does NOT add path-hardening |
| T-03-12 | Tampering | DAP stdio not piped silently proceeding | mitigate | ok_or_else + ? ensures DAP start fails loudly via ShowMessage; the DAP session is not started with a broken stdio channel |
| T-03-13 | Information Disclosure | err.to_string() in ShowMessage for DAP failure | accept | Local editor only; DAP error messages reveal local filesystem paths to the local user |
| T-03-SC | Tampering | No new package installs | accept | Zero new dependencies; slopcheck not applicable |
</threat_model>
