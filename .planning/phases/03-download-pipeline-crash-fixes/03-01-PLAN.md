<objective>
Atomically migrate all 11 blocking HTTP call sites onto the async tokio runtime and drop the
`reqwest::blocking` Cargo feature. Introduces the `DownloadPipeline` wrapper (RT-03) and replaces
`get_url` (blocking) with `get_url_async` + a sync `block_on` shim (RT-02). All 11 call sites
continue to compile without signature changes.

Purpose: This is the keystone change for Phase 3. The `blocking` feature and the async migration
MUST land in the same commit batch — leaving any `reqwest::blocking` call active inside the ambient
Phase-2 runtime causes a "Cannot start a runtime from within a runtime" panic at the first HTTP
request (STATE.md Critical Pitfall #1).

Output: Updated `lapce-proxy/src/lib.rs`, new `lapce-app/src/download.rs`, updated Cargo.toml,
updated call-site files (`update.rs`, `plugin.rs`, `grammars.rs`, `remote.rs`, `plugin/mod.rs`).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → lapce-proxy get_url_async | HTTP response body is untrusted; may be oversized, corrupted, or adversarially crafted |
| reqwest::Response → zstd/GzDecoder | Bytes buffered in memory before decompression; decompression bomb surface |
| reqwest::Response → std::io::copy | Full-body buffer then copy — denial-of-service if response is unbounded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | lapce-proxy/src/lib.rs get_url_async | accept | No integrity verification in Phase 3 (SHA256 is Phase 4 scope); fail-closed path is preserved — errors propagate as Err, not silently dropped |
| T-03-02 | Denial of Service | resp.bytes() in mod.rs/update.rs/grammars.rs | accept | Full-body buffering accepted per D-03 (minimal-invasive); archives typically &lt;100MB; streaming is Phase 5 scope |
| T-03-03 | Elevation of Privilege | GzDecoder / zstd::Decoder on Cursor::new(body) | mitigate (partial) | Phase 3 converts .unwrap() to ? (CRASH-04 in Plan 03); path-traversal guard (archive.unpack) is SEC-04 / Phase 4 |
| T-03-04 | Information Disclosure | https_proxy env var passed to reqwest::Proxy::all | accept | Scheme validation is SEC-05 / Phase 4; Phase 3 preserves existing behavior; no regression |
| T-03-05 | Spoofing | get_url_async retry loop returns first 2xx regardless of content | accept | Content integrity verification (SHA256) is Phase 4 (SEC-01..03); Phase 3 does not regress existing error surfacing |
| T-03-SC | Tampering | No new npm/pip/cargo installs in this plan | accept | Zero new dependencies; only feature removal from reqwest. Slopcheck not applicable. |
</threat_model>
