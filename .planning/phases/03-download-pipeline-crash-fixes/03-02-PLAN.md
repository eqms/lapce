## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS filesystem events → handle_workspace_fs_event | Background notify watcher delivers events with no user validation; workspace path may be None |
| User RPC → git command arms | User-initiated git operations flow through JSON-RPC; workspace and branch/reference parameters are user-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Denial of Service | handle_workspace_fs_event | mitigate | Guard at 1343 returns early when workspace None — eliminates panic; background handler cannot be triggered by remote attacker directly |
| T-03-07 | Information Disclosure | ShowMessage e.to_string() in git error arms | accept | Git errors reveal local filesystem paths; acceptable for local editor (not a server); error message is shown to the local user who owns the workspace |
| T-03-08 | Tampering | eprintln! → ShowMessage replacement | mitigate | Errors now visible to user; no silent failure path remains in these four arms |
| T-03-09 | Repudiation | git error swallowed silently | mitigate | ShowMessage replaces eprintln! — errors are now surfaced and logged via tracing::error! if added; user can see and act on failures |
| T-03-SC | Tampering | No new package installs | accept | Zero new dependencies; slopcheck not applicable |
</threat_model>
