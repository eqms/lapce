<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ZIP archive entries → filesystem | ZipArchive::extract must never write outside the target directory |
| IPC socket → app process | Single-instance IPC must reject connections that are not the legitimate second instance |
| tracing subscriber → log output | Logging must initialize correctly or editor launches silently without log capture |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ZIP extraction path traversal (CVE-2025-29787) | mitigate | zip 2.4.0 `enclosed_name` validation is built into ZipArchive::extract; regression test zip_slip_traversal_rejected asserts fail-closed — traversal entry produces Err or file not written outside tmpdir |
| T-02-02 | Denial of Service | Stale IPC socket from crashed process | mitigate | Manual std::fs::remove_file before ListenerOptions::create_sync() retained in rewritten listen_local_socket; handles the stale-socket case cross-platform |
| T-02-03 | Spoofing | IPC listener bound to wrong socket path | accept | Socket path from Directory::local_socket() is OS config dir (user-owned); no elevation risk; local machine only |
| T-02-04 | Elevation of Privilege | interprocess 2.x GenericFilePath [ASSUMED] type | mitigate | If GenericFilePath is not the correct type name, compiler error surfaces immediately; fallback is to consult docs.rs/interprocess/2.4.2 — no silent failure path |
| T-02-SC | Tampering | cargo installs during build | mitigate | All packages pre-approved in RESEARCH.md Package Legitimacy Audit; no new packages introduced in this plan |
</threat_model>
